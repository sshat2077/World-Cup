## 🎯 أوّل ما يجب فعله في الجلسة الجديدة

**أنا انتهيت من بناء طبقة الخادم كاملة (الباك-إند). الآن أحتاج بناء الواجهات الأمامية الثلاث:**

### 1. `public/home.html` — الصفحة الرئيسية
- ترحيب + شعار الشركة
- زرّ "تسجيل دخول" يوجّه إلى `/admin`
- ملاحظة أن الموظفين يصلون عبر الروابط المُرسَلة لهم

### 2. `public/admin.html` — لوحة الإدارة (الأهمّ والأكبر)

**شاشات فرعية داخلها:**

#### أ) شاشة تسجيل الدخول
- حقل البريد + كلمة المرور
- POST → `/api/auth/login`

#### ب) لوحة التحكّم بعد الدخول
- ترويسة تعرض اسم المستخدم + دوره + زر خروج
- تبويبات (Tabs):
  - **"إنشاء اختبار جديد"** — نموذج إنشاء دعوة
  - **"قائمة الاختبارات"** — جدول يعرض كل الدعوات + حالتها
  - **"النتائج"** — جدول النتائج + زر "عرض تفصيلي"
  - **"إدارة الصلاحيات"** (للمؤسس فقط) — منح صلاحيات على نتائج محدّدة

#### ج) شاشة إنشاء اختبار
- الاسم + البريد + الهاتف (اختياري) + ملاحظات
- اختيار النوع (MBTI / Big Five / DISC) — بطاقات كبيرة
- خيار "السماح بالإيقاف والإكمال" (checkbox)
- عند الإرسال: يعرض الرابط المُولَّد مع زر نسخ

#### د) شاشة عرض النتيجة الكاملة
- تعرض التفسير التفصيلي + SWOT + توصيات المناصب الخمسة
- زر "منح الاطلاع لمستخدم آخر"

### 3. `public/take.html` — رحلة الموظف
**مشابهة لـ `index.html` الحالي، لكن معدّلة لتستخدم `/api/take/:token/*` بدل `/api/test/:id/*`**

**المنطق:**
1. استخراج token من URL: `window.location.pathname.split('/').pop()`
2. GET `/api/take/:token/info` → عرض ترحيب بالموظف
3. GET `/api/take/:token/questions` → استرجاع الأسئلة + التقدّم المحفوظ إن وُجد
4. (إن سُمِح) زر "حفظ والإكمال لاحقاً" يستدعي `POST /api/take/:token/save`
5. في النهاية: `POST /api/take/:token/submit` → عرض النتيجة المختصرة

### 3. المسارات الجاهزة (API Reference)

**المصادقة:**
| Method | Path | الوصف |
|--------|------|-------|
| POST | `/api/auth/login` | `{email, password}` → `{user}` |
| POST | `/api/auth/logout` | يحتاج جلسة |
| GET | `/api/auth/me` | يحتاج جلسة |
| POST | `/api/auth/change-password` | `{current_password, new_password}` |

**الإدارة (تحتاج صلاحية admin أو founder):**
| Method | Path | الوصف |
|--------|------|-------|
| POST | `/api/admin/invitations` | إنشاء دعوة اختبار |
| GET | `/api/admin/invitations` | قائمة الدعوات |
| POST | `/api/admin/invitations/:id/cancel` | إلغاء |
| GET | `/api/admin/results` | قائمة النتائج |
| GET | `/api/admin/results/:id` | نتيجة كاملة |
| POST | `/api/admin/results/:id/grant` | منح اطلاع |
| GET | `/api/admin/users` | قائمة المستخدمين |

**رحلة الموظف (بدون مصادقة، فقط token):**
| Method | Path | الوصف |
|--------|------|-------|
| GET | `/api/take/:token/info` | معلومات الاختبار |
| GET | `/api/take/:token/questions` | الأسئلة + تقدّم محفوظ |
| POST | `/api/take/:token/save` | حفظ جزئي |
| POST | `/api/take/:token/submit` | التسليم النهائي |

## 📝 القرارات التصميمية المعتمدة (مرجعية)

| البند | القرار |
|------|--------|
| **مدّة صلاحية الرابط** | 7 أيام |
| **خيار الإيقاف والإكمال** | اختياري، يُحدّده المدير عند إنشاء الاختبار |
| **المصادقة** | بريد + كلمة مرور + bcrypt (10 rounds) + express-session (8 ساعات) |
| **قاعدة البيانات** | SQLite ملف واحد (`database.sqlite`) عبر `sql.js` (WebAssembly) |
| **عدد المستخدمين المُصرَّح لهم** | 2 افتراضياً (مؤسس + مدير)، يمكن إضافة غيرهم |
| **صلاحيات المؤسس** | كاملة — يرى كل النتائج، يُنشئ اختبارات، يمنح صلاحيات |
| **صلاحيات المدير** | إنشاء اختبارات + رؤية ما أنشأه هو + ما مُنح له |
| **النتيجة المختصرة للموظف** | فقط على الشاشة بعد التسليم (لا تُرسَل له بالبريد) |
| **النتيجة الكاملة** | للإدارة عبر لوحة التحكّم (لاحقاً بالبريد في المرحلة 3) |
| **منع إعادة التسليم** | بعد `status = 'completed'` يُرفَض أي تسليم جديد |

## 🛑 ملاحظات حرجة للجلسة الجديدة

### 1. لا تلمس محركات الحساب
الملفات `server/engines/*-engine.js` مختبرة بـ 122 اختبار ناجح. لا حاجة للتعديل.

### 2. لا تعدّل قاعدة البيانات الدلالية
الملفات في `data/` كاملة ومُراجَعة. إضافة أسئلة = يجب تعديل المحركات + الاختبارات.

### 3. الحقول الحسّاسة مُخفاة أصلاً
الـ API يُرجع فقط `{id, text}` للأسئلة — لا يظهر `axis` / `direction` / `style` / `dimension`. هذا مقصود لمنع الموظف من التلاعب.

### 4. الاختبارات E2E تُمسح DB ثم تعيد إنشاءها
عند تشغيل `test-phase1.js` — سيمسح `database.sqlite` الحالية. احفظ نسخة إن كنت تُجرّب عليها.
